34 /*
35  * try to put a UBlox into binary mode. This is in two parts. First we
36  * send a PUBX asking the UBlox to receive NMEA and UBX, and send UBX,
37  * with a baudrate of 38400. Then we send a UBX message setting rate 1
38  * for the NAV_SOL message. The setup of NAV_SOL is to cope with
39  * configurations where all UBX binary message types are disabled.
40  * 0001: UBX
41  * 0002: NMEA
42  * 0003: NMEA + UBX
43  */

49 /*
50 Structure of an UBlox binary message
51 
52 PREAMBLE1 PREAMBLE2 CLASS MSG_ID LENGTH1 LENGTH2 MESSAGE CHECKSUMA CHECKSUMB
53 1 byte 1 byte 1 byte 1 byte Lowest byte Highest byte LENGTH size 1 byte 1 byte
54 
55 The information is received in the Little endian format (least significant byte first).
56 */
57

61 /*
62 Name Class Description
63 NAV 0x01 Navigation Results: Position, Speed, Time, Acc, Heading, DOP, SVs used
64 RXM 0x02 Receiver Manager Messages: Satellite Status, RTC Status
65 INF 0x04 Information Messages: Printf-Style Messages, with IDs such as Error, Warning, Notice
66 ACK 0x05 Ack/Nack Messages: as replies to CFG Input Messages
67 CFG 0x06 Configuration Input Messages: Set Dynamic Model, Set DOP Mask, Set Baud Rate, etc.
68 MON 0x0A Monitoring Messages: Comunication Status, CPU Load, Stack Usage, Task Status
69 AID 0x0B AssistNow Aiding Messages: Ephemeris, Almanac, other A-GPS data input
70 TIM 0x0D Timing Messages: Timepulse Output, Timemark Results
71 */
